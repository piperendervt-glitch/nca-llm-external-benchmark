## ベンチマークの入手・変換方法

### αNLI

```
入手:
  HuggingFace: allenai/alphanli
  または: https://leaderboard.allenai.org/anli

形式:
  obs1: 「最初の観察」
  obs2: 「最後の観察」
  hyp1: 「仮説1」
  hyp2: 「仮説2」
  label: 1 or 2（正解の仮説番号）

変換方法:
  各タスクから2問を生成:
    問題A: hyp1を提示
           label=1ならCORRECT・label=2ならINCORRECT
    問題B: hyp2を提示
           label=2ならCORRECT・label=1ならINCORRECT

  = 全タスクでCORRECT/INCORRECTが
    必ず50/50になる設計

  変換後にランダムシャッフル（seed=42）
  1092問をサンプリング
```

### RuleTaker-depth-1

```
入手:
  HuggingFace: allenai/ruletaker
  または: https://allenai.org/data/ruletaker

深さ:
  depth-1のみ使用
  （depth-2以上は形式論理型リスクが高い）

形式:
  context: 「ルールセット」
  question: 「命題」
  label: True or False

変換方法:
  そのままCORRECT/INCORRECTに対応
  context + questionを結合してquestionフィールドに

ランダムに1092問をサンプリング（seed=42）
```

## Limitationsの事前記載

```
① 指標の自己循環（問題2）:
  split率・CFR・cal_errorは
  NCAシステムの出力から計算しており
  測定ツールと測定対象が同一システム

② モデル選択（問題3）:
  7b_het_v1（qwen2.5/llama3/mistral）は
  Ollamaで入手しやすいモデルを選択した
  7b_het_v2（qwen2.5/gemma2/phi3）との
  比較で部分的に検証する

③ 温度設定（問題5）:
  temperature=0.7固定
  他の設定での検証は今後の課題

④ COPA除外（問題4）:
  COPAは500問のみでpower不足（必要1092問）
  のため除外した

⑤ 探索フェーズの指標事後選択（問題6）:
  CFR・split率・cal_errorは
  探索フェーズ（MVE-01〜09）の結果を見てから
  採用した
  本実験ではPre-registrationで事前固定する
```
