```
□ 「きりが良い数だから」→ power analysisが必要

□ 「とりあえず動くものを」→ 外部妥当性の確認が必要
```

## ⓪ 意思決定フロー（設計上の重要判断時）

設計上の判断でClaudeとGrokの意見が分かれた場合:

```
Step 1: Claude + Grokに聞く
  一致 → 採用

Step 2: 分かれた → ChatGPTに聞く
  多数決（2対1）→ 多数派を採用

Step 3: 3者全員異なる → Geminiに聞く
  多数決で採用

Step 4: 4者全員異なる → 人間（Robosheep）が判断
```
